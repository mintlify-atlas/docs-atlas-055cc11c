This page explains the full execution flow of the Kahoot Answer Bot, from joining a game to submitting answers.

## Full flow

<Steps>
  <Step title="Pin validation">
    The bot sends a `GET` request to verify the game pin and obtain session credentials:

    ```
    GET https://play.kahoot.it/reserve/session/{pin}/?{timestamp}
    ```

    A successful response (HTTP 200) returns:
    - `x-kahoot-session-token` header — a base64-encoded token used in the next step
    - A JSON body containing a `challenge` field — a JavaScript snippet that must be evaluated

    If the pin is invalid or the game has not started, the bot exits with an error.
  </Step>

  <Step title="Challenge solving">
    Kahoot uses a client-side JavaScript challenge to derive the session ID. The bot solves this without a browser by running the challenge in **`py_mini_racer`**, a Python-embedded V8 engine.

    The solving process:
    1. The raw challenge string is cleaned and restructured into a callable JS function.
    2. `py_mini_racer.MiniRacer` evaluates the function and returns a solution string.
    3. The solution is XOR'd character-by-character with the base64-decoded session token:

    ```python
    decodedToken = base64.b64decode(self.sessionToken).decode('utf-8', 'strict')
    solChars = [ord(s) for s in solution]
    sessChars = [ord(s) for s in decodedToken]
    sessionID = "".join([chr(sessChars[i] ^ solChars[i % len(solChars)]) for i in range(len(sessChars))])
    ```

    The resulting `sessionID` is used to construct the WebSocket URL.
  </Step>

  <Step title="Answer lookup">
    Before or at the start of the game, the bot fetches answers from the Kahoot API:

    ```
    GET https://create.kahoot.it/rest/kahoots/{quizID}
    ```

    For each question in the response, the bot reads the `choices[].correct` field to identify the correct answer index.

    Only questions of type `quiz` or `multiple_select_quiz` are processed. Other content types (slides, videos, etc.) are skipped and a placeholder index is used.

    <Note>
      Answer lookup requires authentication. Pass your Kahoot credentials with `-e` and `-a`, or supply the quiz UUID directly with `-i`.
    </Note>
  </Step>

  <Step title="WebSocket connection">
    The bot opens a WebSocket connection using the **CometD** protocol via `aiocometd`:

    ```
    wss://play.kahoot.it/cometd/{pin}/{sessionID}
    ```

    CometD is a publish/subscribe messaging protocol layered on top of WebSockets. The bot subscribes to three channels:

    | Channel | Purpose |
    |---|---|
    | `/service/controller` | Send actions (login, answers) |
    | `/service/player` | Receive player-facing events |
    | `/service/status` | Receive game status events |
  </Step>

  <Step title="Game loop">
    After subscribing, the bot sends a login message to `/service/controller`:

    ```json
    {
      "host": "kahoot.it",
      "gameid": "<pin>",
      "captchaToken": "KAHOOT_TOKEN_eyJ2ZXJzaW9uIjoiIn0=",
      "name": "<nickname>",
      "type": "login"
    }
    ```

    It then enters an async `for` loop, consuming messages from the server. Each incoming message carries a numeric `id` field that maps to a game event (see the [message ID table](#message-id-lookup-table) below).
  </Step>

  <Step title="Answering questions">
    When the bot receives a `START_QUESTION` event (message ID 2), it:
    1. Reads `data['questionIndex']` to identify the current question.
    2. Looks up the correct answer index from the pre-fetched answers list.
    3. Publishes the answer to `/service/controller` with message ID 45:

    ```json
    {
      "choice": 0,
      "meta": {
        "lag": 0,
        "device": {
          "userAgent": "kbot",
          "screen": {"width": 1920, "height": 1080}
        }
      }
    }
    ```

    The `choice` field is a zero-based index mapping to answer positions:

    | Index | Color |
    |---|---|
    | 0 | RED |
    | 1 | BLUE |
    | 2 | YELLOW |
    | 3 | GREEN |

    If no answers were fetched, `DEFAULT_ANSWER = 1` (BLUE) is submitted as a fallback.
  </Step>
</Steps>

## Question type filtering

The bot only processes questions of these types:

```python
allowedTypes = ['quiz', 'multiple_select_quiz']
```

Slides, videos, open-ended questions, and other content types are not in `allowedTypes`. When the bot encounters them during answer pre-fetching, it skips them and inserts a placeholder so the answer index list stays aligned with the question sequence.
